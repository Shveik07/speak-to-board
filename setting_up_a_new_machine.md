# Подготовка локальной машины — Создание SSH-ключа
Сгенерируйте пару ключей. Вставьте и выполните команду:
```
ssh-keygen -t ed25519 -C "email@example.com"
-t ed25519 — указывает тип ключа (современный и безопасный) .

-C — комментарий (можно указать email, чтобы помнить, для чего ключ).
```

# Первое подключение к серверу (логин и пароль)
Сейчас вы подключитесь к серверу, используя данные, которые прислал хостинг-провайдер.

```
ssh root@<IP-адрес-сервера>
```
# Копируем ваш открытый ключ на сервер (пока от root)
Самый простой способ — использовать ssh-copy-id.

```
ssh-copy-id -i C:\Users\Имя_Пользователя\.ssh\id_ed25519.pub root@<IP-адрес-сервера>
-i — указывает путь к вашему публичному ключу.

root@... — пользователь и сервер, куда копируем.
```

# Проверка входа по SSH-ключу (для root)

```
ssh root@<IP-адрес-вашего-сервера>
```

# Создаём нового пользователя (для повседневной работы)
```
adduser myuser
```
# Дайте пользователю права администратора (добавьте в группу sudo):
```
usermod -aG sudo myuser
```
# Копируем SSH-ключ новому пользователю
```
bash
# 1. Создаём папку .ssh в домашней папке нового пользователя
mkdir -p /home/myuser/.ssh

# 2. Копируем содержимое файла authorized_keys от root к новому пользователю
rsync --archive --chown=myuser:myuser /root/.ssh /home/myuser/
Команда rsync скопирует всю папку .ssh, сохранив права доступа и назначив владельцем нового пользователя .
```

Для надёжности можно явно проверить права доступа:
```

chmod 700 /home/myuser/.ssh
chmod 600 /home/myuser/.ssh/authorized_keys
```

# Финальная настройка безопасности
Теперь сделаем сервер более защищённым .

Отключить вход для пользователя root по SSH.

```
sudo nano /etc/ssh/sshd_config
строку #PermitRootLogin yes (или PermitRootLogin without-password). Изменить её на:


PermitRootLogin no

Отключить вход по паролю. строку #PasswordAuthentication yes. Изменить её на:


PasswordAuthentication no
```

# Перезапустить службу SSH, чтобы изменения вступили в силу:

```
sudo systemctl restart ssh
или


sudo systemctl restart sshd
.
```

Настройть простой файрвол (UFW), разрешив только SSH (порт 22, если его не меняли).

```
# Разрешить SSH
sudo ufw allow 22/tcp
# Включить файрвол
sudo ufw --force enable
# Посмотреть статус
sudo ufw status verbose
```

# Откройте редактор sudoers с помощью visudo
Это самая безопасная команда для редактирования. Она блокирует файл от одновременных изменений и проверяет синтаксис перед сохранением, что критически важно .

Выполните команду для создания отдельного файла конфигурации для вашего пользователя:

```
sudo visudo -f /etc/sudoers.d/myuser-nopasswd
Вместо myuser-nopasswd можно использовать любое имя, состоящее из букв, цифр, дефисов и знаков подчёркивания. Важно: в имени файла не должно быть точек (.), иначе он будет проигнорирован системой .
```
# Добавьте правило NOPASSWD
```
В открывшемся редакторе (обычно это nano) добавьте следующую строку :


myuser ALL=(ALL) NOPASSWD: ALL
Пояснение строки:

myuser — имя вашего пользователя.

ALL=(ALL) — правило действует с любого хоста и позволяет запускать команды от имени любого пользователя.

NOPASSWD: ALL — ключевая директива, отключающая запрос пароля для всех команд .

Если вы хотите сохранить пароль для большинства команд и убрать его только для некоторых (например, для перезапуска служб), можно использовать более тонкую настройку . Например, для systemctl и apt:


myuser ALL=(ALL) NOPASSWD: /usr/bin/systemctl, /usr/bin/apt
Шаг 4: Сохраните файл и выйдите

visudo автоматически проверит синтаксис. Если будет ошибка, он не даст сохранить файл и предложит исправить её .
```

# Убедитесь, что у файла правильные права доступа
```
Файлы в /etc/sudoers.d/ должны иметь права 0440. Скорее всего, visudo выставит их автоматически, но для надёжности можно проверить и исправить :


sudo chmod 0440 /etc/sudoers.d/myuser-nopasswd
```
# Проверьте результат

Закройте текущую SSH-сессию (exit) и зайдите заново, чтобы изменения гарантированно применились . Затем выполните любую команду с sudo, например:
```
sudo whoami
```
Если команда мгновенно вернула root, не спрашивая пароль — всё настроено правильно. Также можно посмотреть список своих sudo-прав командой sudo -l — в выводе вы увидите запись NOPASSWD .

# Как откатить изменения

Просто удалите созданный файл конфигурации:

sudo rm /etc/sudoers.d/myuser-nopasswd

Или закомментируйте строку в этом файле, добавив в начало символ #.
