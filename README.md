# 🎤 Speak-to-Board — голосовое создание задач в Trello
🤖 это Telegram-бот, который превращает голосовые сообщения в задачи на доске Trello. Бот использует локальное распознавание речи (Vosk) для полной автономности и не требует платных API. Подходит для личного использования и небольших команд.

**Voice to Kanban Bot** — это умный Telegram-бот, который принимает голосовые сообщения, распознает речь с помощью ИИ и автоматически создает задачи на вашей канбан-доске (Trello/собственная доска).

## ✨ Возможности

- 🎙️ **Голосовой ввод** — отправляйте голосовые сообщения вместо набора текста
- 🤖 **AI-распознавание** — точная транскрибация речи с помощью Vosk
- 📋 **Автосоздание задач** — каждая голосовая заметка становится карточкой на доске
- 🧠 **Умный анализ** (в разработке) — извлечение дедлайнов, приоритетов и тегов из речи

### Описание потоков данных

1. Пользователь → Telegram: отправляет голосовое сообщение боту.

2. Telegram Bot API → сервер: через long polling (или webhook) бот получает обновление.

3. Основной обработчик (aiogram):
   * Скачивает голосовой файл через bot.download_file() во временную папку temp/.
   * Передаёт путь к файлу в модуль audio_processor.

4. Модуль audio_processor:
    * Конвертирует OGG в WAV с помощью FFmpeg (параметры: 16 кГц, моно, 16 бит).

5. Модуль vosk_recognizer:
    * Загружает модель Vosk (однократно при старте).
    * Читает WAV-файл и возвращает распознанный текст.

6. Модуль trello_api:
    * Формирует POST-запрос к Trello API с ключами, ID списка, названием и описанием задачи.
    * Отправляет запрос и обрабатывает ответ.

7. Trello создаёт карточку в указанном списке.

8. Бот отправляет пользователю подтверждение со ссылкой на карточку.

9. Временные файлы удаляются (функция cleanup_temp_files).

### Инфраструктурные компоненты

* Flask-сервер работает в том же процессе, но в фоновом потоке, предоставляя эндпоинты / и /health для мониторинга (на хостингах  вроде Render это предотвращает «засыпание»).
* Systemd управляет жизненным циклом процесса: запускает при старте системы, перезапускает при падении, логирует stdout/stderr.
* Файл .env хранит секреты (токены, ключи), не попадая в репозиторий.

## 📦 Установка

### Локальный запуск

1. Клонируйте репозиторий:
   ```
    git clone https://github.com/Shveik07/speak-to-board.git
    cd speak-to-board
   ```

2. Создание виртуального окружения и установка зависимостей
```
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
```

3. Установите зависимости:
    ```
    pip install -r requirements.txt
    ```


3. Создайте файл .env:
    ```
    BOT_TOKEN=ваш_токен_от_botfather
    KANBAN_API_URL=https://api.trello.com/1/cards
    KANBAN_API_KEY=ваш_api_ключ
    KANBAN_API_TOKEN=ваш_api_токен
    LIST_ID=id_колонки_на_доске
    ```

4. Запустите бота:
    ```
    python bot.py
    ```

### ☁️ Развёртывание на сервере (Ubuntu + systemd)

1. Перенос проекта на сервер
   Скопируйте файлы через scp или rsync, либо клонируйте репозиторий напрямую на сервере и повторите шаги 2–5 (установка
   зависимостей, модели, настройка .env).

2. Настройка автозапуска через systemd
```
    Создайте файл /etc/systemd/system/voice-bot.service:


    ini
    [Unit]
    Description=Voice to Kanban Bot
    After=network.target

    [Service]
    User=shveik                               # замените на своего пользователя
    WorkingDirectory=/home/shveik/projects/speak-to-board
    ExecStart=/home/shveik/projects/speak-to-board/venv/bin/python app.py
    Restart=always
    RestartSec=10

    [Install]
    WantedBy=multi-user.target
```

3. Обновите конфигурацию и запустите службу:
```
    sudo systemctl daemon-reload
    sudo systemctl enable voice-bot.service
    sudo systemctl start voice-bot.service
    sudo systemctl status voice-bot.service
```

4. Просмотр логов
```
    sudo journalctl -u voice-bot.service -f
```
